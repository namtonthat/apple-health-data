# Health Dashboard (web)

A mobile-first static dashboard that consolidates the Streamlit views (Recovery,
Nutrition & Body, Exercises, Performance Insights) into one app:
**Overview · Recovery · Body · Training**.

- **Stack:** Next.js 14 (static export) + Tremor + Tailwind, deployed on Vercel.
- **Data:** bundled at build time from `web/data/dashboard.json`, which is
  generated by the Python pipeline (`uv run python run.py export-web`) reading the
  same S3 `transformed/` parquet the Streamlit app uses. No server, no AWS creds
  in the web layer — the JSON is committed and the site is rebuilt on each push.

## Local development

```bash
# 1. Regenerate the data snapshot (from the repo root, needs .env / AWS creds)
uv run python run.py export-web

# 2. Run the app
cd web
npm install
npm run dev          # http://localhost:3000
npm run build        # static export to web/out/
```

## Data flow

```
S3 transformed/ parquet
  → run.py export-web  →  web/data/dashboard.json  (committed)
  → next build (static export)  →  Vercel (public)
```

The daily GitHub Actions workflow (`.github/workflows/refresh-data.yml`)
regenerates `web/data/dashboard.json` after each refresh and commits it; the push
triggers a Vercel redeploy.

## Deploy to Vercel (one-time setup)

1. Import the repository at <https://vercel.com/new>.
2. Set **Root Directory** to `web`.
3. Framework preset auto-detects **Next.js**; leave build/output defaults.
4. Deploy. Every push that changes `web/` (including the daily data commit)
   redeploys automatically.

No environment variables are needed — all data is baked into the committed JSON.
